# Week 1 — Product Immersion and Setup

2-hour in-person session at Studio School, led by Oliver + Tatenda. Tony joins via Zoom for 30 minutes.

No code this week. The goal is to understand what Beat Modules does, meet the platform, and get your development environment set up so Week 2 isn't spent fighting installers.

## In the Session

### Hour 1 — Who we are, what we build

- Oliver introduces Beat Modules and the team
- Guided walkthrough of the Beat Modules platform (student view + teacher view)
- Tony joins via Zoom to explain the tech stack (React front end, Go back end, how they fit together)

### Hour 2 — Explore the platform

Split into groups:

- One group works through the **vaping module**
- The other through **positive masculinities**

Play the games, complete activities, review the full student experience.

Each student writes down:

- Three features you liked
- Three things you'd improve
- One feature you'd want to build yourself (save this — you'll refer back to it in Weeks 6-8)

Tatenda is on hand for any technical questions.

## Independent Work (~3 hours)

Two things — both essential before Week 2.

### 1. Install your development tools

Follow the setup guide for your operating system (they're at the repo root):

- [`../SETUP-MAC.md`](../SETUP-MAC.md)
- [`../SETUP-WINDOWS.md`](../SETUP-WINDOWS.md)

By the end you'll have:

- VS Code
- Git
- Node.js + npm
- Go

Don't skip the "verify it works" steps. If `npm install && npm run dev` doesn't run the Week 2 starter successfully by Monday, ping the group chat.

### 2. Pre-reading for Week 2 (React basics)

See [`PRE-READING.md`](PRE-READING.md) — a couple of short videos and some React docs to skim. You don't need to memorise React — just recognise what "component", "props" and "state" mean when Eman says them.
