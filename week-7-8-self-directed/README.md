# Weeks 7-8 — Self-Directed Build + Polish

No in-person session. 1-hour online drop-in each week. Async group chat available throughout.

By the start of Week 7 you should have a working full-stack prototype. These two weeks are about making it *good*, then making it *presentable*.

## Week 7 — Feature Development (w/c 23 June)

Add things that make the game richer. Pick one or two from the list below — don't try all of them:

- **Leaderboard** — use `fetchResults()` in the React client to show every player's scores. Sort by highest percentage.
- **Difficulty levels** — group your content into tiers; let the player pick easy/medium/hard at the start.
- **More content** — the single biggest improvement most games can make. Double your content count.
- **Results summary** — instead of just a final score, show which questions were answered right/wrong and why.
- **Player name input + history** — store names so returning players can see their past scores.
- **Second game mode** — if your first mode is multiple-choice, add a scenario mode (or vice versa).

1-hour online drop-in with the dev team for troubleshooting. Show up with a specific problem if you have one.

## Week 8 — Polish and Presentation Prep (w/c 30 June)

Stop adding features. Start polishing:

- **UI polish** — colours, fonts, spacing, animations. Make it feel finished, not a prototype.
- **Bug fixing** — play your game 10 times end-to-end. What breaks? Fix it.
- **User flow tidy-up** — no dead ends, no confusing screens, clear feedback at every step.
- **Content review** — read every prompt and every bit of feedback text. Does it teach? Is it age-appropriate? Is it accurate on anti-racism?
- **5-minute pitch prep** — for the Week 9 showcase. Cover: what your game teaches, how it plays, one thing you're proud of, one thing you'd do with more time.

1-hour online drop-in + 15-minute presentation prep with the dev team. Bring your slides (or demo plan) to the drop-in.

## Showcase Checklist (for Week 9)

- [ ] Game runs reliably end-to-end — no crashes in a cold start + full playthrough
- [ ] UI looks intentional, not default template
- [ ] All content is final — no placeholder text
- [ ] Pitch is 5 minutes or less, timed at least once
- [ ] Laptop ready, charger packed, backup of your code somewhere cloud (Git push!)

## Getting Help

- Async group chat for quick questions and screenshots
- Online drop-in (scheduled weekly) for longer debugging
- If something's broken and you're blocked — post it in the chat straight away, don't wait till the drop-in
